# C2 : Resources

_Describe the resources for the API_

A API disponibiliza x recursos principais (restaurante, ingredientes), com as respetivas operações CRUD essenciais: post, get, put e delete.

## 1 - Restaurantes
Faz a gestão dos restaurantes do sistema.

**Operações CRUD:**

- GET
- GET
- POST
- PUT
- DELETE

**Campos:**

- 'nome' (obrigatório)
- ...

## 2 - Pratos
Faz gestão dos pratos pedidos dos restaurantes.

**Operações CRUD:**

- GET
- GET
- POST
- PUT
- DELETE

**Campos:**

- 'restaurante_id' (obrigatório)
- ...

## 3 - Clientes
Faz gestão dos clientes do sistema.

**Operações CRUD:**

- GET
- GET
- POST
- PUT
- DELETE

**Campos:**

- 'nome' (obrigatório)
- ...

## 4 - Pedidos
Faz gestão dos pedidos do sistema.

**Operações CRUD:**

- GET
- GET
- POST
- PUT
- DELETE

**Campos:**

- 'nome' (obrigatório)
- ...

##Relação entre os Recursos

- **restaurantes -> pratos**: Um restaurante tem vários pratos
- **clientes -> pedidos**: Um cliente pode fazer vários pedidos
- **restaurantes -> pedidos**: Um restaurante recebe vários pedidos

### Relação m:n
- **pratos <-> ingredientes**: Um prato pode ter vários ingredientes, um ingrediente pode estar em vários pratos (via tabela `pratos_ingredientes`)
