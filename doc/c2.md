# C2 : Resources

A API disponibiliza 6 recursos principais: Restaurantes, Pratos, Clientes, Ingredientes, Entregas e Pedidos; cada um com as respetivas operações CRUD essenciais: post, get, put e delete.

## 1 - Restaurantes
Faz a gestão dos restaurantes do sistema.

**Estrutura do Recurso:**
(json no open api, tipo os exemp no yaml)

**Operações CRUD:**

- GET /api/restaurantes -> listar todos os restaurantes
- GET /api/restaurantes/{id} -> listar restaurante pelo seu id
- POST /api/restaurantes -> criar restaurante
- PUT /api/restaurantes/{id} -> atualizar informações do restaurante
- DELETE /api/restaurantes/{id} -> apagar restaurante pelo seu id

**Campos:**

- id (obrigatório, primary key)
- nome (obrigatório)
- morada (obrigatório)
- telefone
- especialidade
- tempo médio de preparação
- taxa de entrega
- (...)

## Relação entre os Recursos

**Relações 1-n**
- **restaurantes -> pratos**: Um restaurante tem vários pratos
- **clientes -> pedidos**: Um cliente pode fazer vários pedidos
- **restaurantes -> pedidos**: Um restaurante recebe vários pedidos
- **restaurantes -> entregas**: Um restaurante faz várias entregas

**Relação m-n**
- **pratos <-> ingredientes**: Um prato pode ter vários ingredientes e um ingrediente pode estar em vários pratos (via tabela `pratos_ingredientes`)
