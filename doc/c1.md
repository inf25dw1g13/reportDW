# C1 : Food Delivery API

## Descrição
Este trabalho tem como objetivo fazer uma interação entre uma aplicação, api, que através de métodos post, get, put e delete, interage com a sua database, simulando uma plataforma de entregas de comida.

O projeto consiste-se numa api que faz simulação da criação do(s) pedido(s) http, obtenção de informação sobre o(s) mesmo(s), bem como a deleção. Além disso, utiliza-se uma base de dados MariaDB para estrutura e armazenamento de dados relacionados às entregas com as seguintes tabelas principais:
- Restaurantes, Pratos, Clientes, Pedidos, Ingredientes e Entregas.

A base de dados foi preenchida com dados suficientes para demonstração, contendo mais de 40 registos por tabela , garantindo um volume adequado de dados para a apresentação do trabalho.

Inclui também um MaxScale como middleware entre as 3 base de dados, sendo que destas, 2 são apenas para leitura (Réplicas) e 1 para escrita (Master).
O MaxScale controla este fluxo: se for enviado um método de escrita, é redirecionado para a base de dados Master, se for um método de leitura, é redirecionado para as Réplicas.
